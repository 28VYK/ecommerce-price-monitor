# Contributing to E-commerce Price Monitor

Thank you for considering contributing! Here are some guidelines.

## How to Contribute

### Reporting Bugs
- Use GitHub Issues
- Include Python version, OS, and error messages
- Provide steps to reproduce

### Suggesting Features
- Open an issue with the "enhancement" label
- Describe the use case
- Explain expected behavior

### Code Contributions

1. **Fork the repository**
2. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow existing code style
   - Add comments for complex logic
   - Update README if needed

4. **Test your changes**
   ```bash
   python price_monitor.py
   ```

5. **Commit with clear messages**
   ```bash
   git commit -m "Add: feature description"
   ```

6. **Push and create PR**
   ```bash
   git push origin feature/your-feature-name
   ```

## Code Style

- Follow PEP 8
- Use meaningful variable names
- Add docstrings to functions
- Keep functions focused and small

## Testing

Before submitting:
- [ ] Code runs without errors
- [ ] All features work as expected
- [ ] No hardcoded personal data
- [ ] Config examples are generic

## Areas We Need Help

- [ ] Support for more e-commerce platforms
- [ ] Alternative notification methods (email, Discord, Slack)
- [ ] Web UI for configuration
- [ ] Docker containerization
- [ ] Unit tests
- [ ] Documentation improvements

## Questions?

Open an issue for discussion before starting major changes.
